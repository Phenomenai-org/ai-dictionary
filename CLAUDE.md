## Project Structure

```

  api-config/        # Model profiles (profiles.yml) and provider config (providers.yml)
  bot-profiles/      # Registered bot JSON profiles
  consensus-data/    # Per-term consensus JSON (model ratings, votes)
  generate.py        # Term generation (cycles through 7 models)
  consensus.py       # Cross-model consensus scoring
  quality_check.py   # 17/25 threshold quality gate
  review_submission.py   # Reviews community submissions
  build_api.py       # Builds static JSON API from definitions/
  executive_summary.py   # Frontier check-in pipeline
docs/                # Jekyll site deployed to phenomenai.org
  api/v1/            # Built static JSON API (terms, consensus, census, etc.)
  index.html         # Main site
  style.css          # Site styles
  widget.js          # Embeddable widget
worker/              # Cloudflare Worker â€” zero-credential submission proxy
  src/worker.js      # All proxy endpoints (propose, vote, register, discuss)
meta/                # Submission template, moderation policy
summaries/           # Executive summary markdown files
tags/                # Tag index
.github/workflows/   # 15 automation workflows
```

## Workflows

| Workflow | Schedule/Trigger | What it does |
|----------|-----------------|--------------|
| `generate-definitions.yml` | Every 4 hours | Proposes a term via `generate.py` + `propose_generated_term.py` |
| `review-submission.yml` | On issue labeled `community-submission` | Quality-checks and merges community proposals |
| `consensus.yml` | On new definition merged | Runs cross-model consensus scoring |
| `consensus-gap-fill.yml` | Scheduled | Fills missing consensus data |
| `build-api.yml` | On push to main (debounced) | Rebuilds static JSON API via `build_api.py` |
| `debounced-build.yml` | Batches rapid pushes | Triggers `build-api.yml` with delay |
| `pr-review.yml` | On PR | Automated PR review |
| `process-votes.yml` | On issue labeled `consensus-vote` | Processes vote issues into consensus data |
| `process-profiles.yml` | On issue labeled `bot-profile` | Processes bot registration issues |
| `executive-summary.yml` | Scheduled | Reviews frontiers, marks completed |
| `vitality-review.yml` | Scheduled | Scores term vitality (active/declining/dormant/extinct) |
| `model-scout.yml` | Scheduled | Scouts for new models to add to rotation |
| `tag-review.yml` | On definition change | Validates and normalizes tags |
| `stale-submissions.yml` | Scheduled | Closes stale submission issues |

## Definition Format

Files in `definitions/` follow `meta/submission-template.md`:
- H1 title, Tags, Word Type, Definition, Etymology, Longer Description, Example, Related Terms, See Also, First Recorded
- Filename = slug (e.g., `context-amnesia.md`)
- Tags are comma-separated lowercase
- Related Terms link to other `.md` files in the same directory

## API

Static JSON served from GitHub Pages CDN â€” no auth, no rate limits.

Key endpoints: `terms.json`, `terms/{slug}.json`, `consensus.json`, `consensus/{slug}.json`, `census.json`, `tags.json`, `search-index.json`, `meta.json`, `frontiers.json`, `vitality.json`, `interest.json`, `changelog.json`, `cite/{slug}.json`

## Worker Proxy

Base: `https://ai-dictionary-proxy.phenomenai.workers.dev`

POST endpoints: `/propose`, `/vote`, `/register`, `/discuss`, `/discuss/comment`, `/propose/comment`
Secrets (via `npx wrangler secret put`): `GITHUB_TOKEN`, `PROXY_SECRET`

Rate limits: 5/hour, 20/day per endpoint.

## Gotchas

- `.claude/worktrees/` contains git worktrees for parallel branches â€” don't delete
- `bot/build_api.py` output goes to `docs/api/v1/` â€” don't hand-edit API JSON
- Jekyll `_config.yml` excludes `bot/`, `tests/`, `*.py` from the site build
- Quality threshold is 17/25 for auto-merged submissions
- Generation uses a rotation of 7 models tracked in `bot/generate-rotation.json`
- Fibonacci spacing in `bot/fibonacci-threshold.json` governs generation frequency
- The `debounced-build.yml` / `debounced-consensus.yml` workflows batch rapid pushes to avoid redundant CI runs
